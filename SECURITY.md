## 报告安全漏洞

如果你发现了安全漏洞，请**不要**在公开的 issue 中报告。

### 报告流程

1. **发送邮件**: 将漏洞详情发送到 me@zhaifanhua.com
2. **包含信息**:
   - 漏洞描述
   - 受影响的版本
   - 重现步骤
   - 潜在影响
   - 建议的修复方案（如果有）

3. **响应时间**:
   - 我们会在 48 小时内确认收到你的报告
   - 我们会在 7 天内评估漏洞并给出初步响应
   - 我们会在 30 天内发布修复版本（取决于漏洞严重程度）

### 漏洞严重级别

我们按照以下标准评估漏洞严重程度：

- **严重 (Critical)**: 可导致远程代码执行、数据泄露的漏洞
- **高 (High)**: 可绕过安全机制、权限提升的漏洞
- **中 (Medium)**: 可导致信息泄露、拒绝服务的漏洞
- **低 (Low)**: 其他安全问题

### 披露政策

- 我们将与你合作，在修复发布后公开披露漏洞
- 我们会在发布说明中致谢报告者（除非你要求匿名）
- 我们鼓励负责任的披露

## 安全最佳实践

### 依赖管理

- 定期更新依赖到最新的安全版本
- 使用 `pnpm audit` 检查已知漏洞
- 监控 GitHub Security Advisories

### 代码审查

- 所有 PR 必须经过代码审查
- 使用 ESLint 和 TypeScript 进行静态检查
- 实施 CI/CD 自动化测试

### 输入验证

- 对所有用户输入进行验证和清理
- 防止 XSS 攻击
- 防止注入攻击
